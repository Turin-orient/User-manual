# Usage Monitoring

Monitor system usage, track metrics, and understand how Neurond Assistant is being used.

## Purpose
Track adoption, identify issues, and optimize system performance.

## Who Can Perform This
✅ Administrators only

---

## Key Metrics to Monitor

### User Activity
- **Active Users** - Daily/weekly/monthly
- **New Users** - Growth rate
- **Engagement** - Messages per user
- **Peak Usage Times** - When system is busiest

### API Usage
- **Token Consumption** - API costs
- **Requests per Day** - Volume
- **Average Response Time** - Performance
- **Error Rate** - Quality metric

### Feature Usage
- **Document Uploads** - Volume and types
- **Knowledge Base Queries** - Search frequency
- **Exports** - How often users export
- **Popular Topics** - What users ask about most

---

## Accessing Analytics

**Via Admin Dashboard:**
1. Go to Admin → Analytics or Reports
2. Select time period (last 7/30/90 days)
3. View metrics and charts

[Screenshot: Analytics dashboard showing usage graphs]

---

## Reports You Can Generate

### Usage Summary Report
- Total users
- Total conversations
- Total messages
- API costs

### User Activity Report
- Per-user statistics
- Department breakdown
- Power users vs occasional users

###API Cost Report
- Spend by service (OpenAI, Azure, Google)
- Cost trends over time
- Projected monthly cost

### Error Report
- System errors frequency
- User-reported issues
- Failed API calls

---

## Setting Up Alerts

**Configure notifications for:**
- API quota approaching limit (e.g., 80%)
- Unusual spike in usage
- Error rate exceeds threshold
- System downtime

---

## Using Insights

**Optimize costs:**
- Identify high-usage periods
- Right-size API tier/plan
- Implement caching if available

**Improve adoption:**
- See which departments use it most
- Identify training opportunities
- Highlight underused features

**Enhance performance:**
- Monitor response times
- Scale resources if needed
- Address bottlenecks
